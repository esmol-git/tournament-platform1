# План: формат турнира «ручной» — без автогенерации расписания

## Статус реализации

- **Фаза 1–2 (MVP):** в Prisma добавлен `TournamentFormat.MANUAL`; в `TournamentsService` запрещены `generateCalendar`, `generateCalendarFromTemplate`, `generatePlayoff`; для `reorderRound` разрешены `SINGLE_GROUP` и `MANUAL`. На фронте: опция формата, скрыты кнопки генерации на странице турнира, пересчёт календаря при смене состава отключён для `MANUAL`.
- **Фаза 3 (ручные матчи):** `POST /tournaments/:id/matches`, `DELETE …/matches/:id` (только `MANUAL`), `PATCH …/matches/:id` — время для любого формата; смена `homeTeamId`/`awayTeamId` только для `MANUAL` (сброс счёта и событий). В админке `[id].vue`: вкладка **«Матчи»** (полный список, протокол, изменить, удалить), плюс календарь с ручным добавлением/удалением.
- **Свободные матчи:** в Prisma у `Match` поле `tournamentId` опционально (`NULL` — матч без турнира). API: `GET/POST/PATCH/DELETE /tenants/:tenantId/standalone-matches`, `PATCH …/protocol`, `POST …/attach` с телом `{ tournamentId }` — только для турнира `MANUAL`, обе команды в **составе** турнира (`TournamentTeam`). Если у турнира **несколько групп** и матч группового этапа (`stage: GROUP`): обе команды должны быть в **одной** группе (как на вкладке «Составы»); при прикреплении без `groupId` на матче сервер **подставляет** `groupId` по составам; если команды в разных группах — ошибка. **`POST /tenants/:tenantId/matches/:matchId/detach`** — открепить матч от турнира `MANUAL` обратно в свободные (пересчёт таблицы). **`GET /tenants/:tenantId/matches`** — список матчей с `tournamentId != null` (query: `tournamentId?`, `page`, `pageSize`). Страница **Админ → Матчи**: вкладки «Свободные» и «В турнирах»; прикрепление удобно для заранее сыгранных матчей, иначе проще создавать матчи сразу в **карточке турнира → Матчи**.
- **Завершённые матчи:** статусы `FINISHED`, `PLAYED`, `CANCELED` — мутации матча и протокола на бэкенде запрещены; в UI действия скрыты/disabled, протокол — только просмотр.
- **Группы в MANUAL:** в карточке турнира задаётся «Кол-во групп» (2–8) и «Команд выходит из группы». Записи `Группа A`… досоздаются при **создании/обновлении турнира** и при каждом **GET** турнира, если их не хватает (раньше они создавались только при нуле матчей — из‑за этого после первого матча вторая группа не появлялась). На вкладке **Составы** — колонки с drag-and-drop, подсказка по ровному делению. В матчах: **номер тура** и **группа**; `PATCH …/matches/:id` — `roundNumber`, `groupId`.
- **Плей-офф в MANUAL:** автогенерация **календаря** по шаблону запрещена (`assertNotManualCalendarFormat`). **Сетка плей-офф по итогам групп** — та же, что для `GROUPS_*`: `POST /tournaments/:id/playoff` → `generatePlayoffGeneric` (турнирные таблицы по группам, топ-K на группу, степень двойки). Условия: все групповые матчи в БД (полный круг или как в валидации) и все с результатами. На фронте: вкладка **«Матчи»** — блок «Сгенерировать плей-офф» для MANUAL с несколькими группами. Отдельно **вручную** можно добавлять матчи на вылет: **«Добавить матч»** → стадия **Плей-офф** (`AdminTournamentMatchesWorkspace.vue`).
- **Группа у матча (несколько групп):** для `MANUAL` с `groupCount > 1` у матча группового этапа **обязателен** `groupId`; обе команды должны состоять в этой группе (проверка в `MatchesService`). Иначе таблица по группам и статистика расходятся с тем, что видно в колонках «Составы».
- **Миграция БД:** `npx prisma migrate deploy` (файл `20260322140000_tournament_format_manual`).

## Идея

Добавить значение **`MANUAL`** в enum `TournamentFormat` (Prisma + API + фронт).

- Турнир с `format = MANUAL` — **только ручные матчи** (создание/редактирование через CRUD матчей).
- **Любая автогенерация расписания запрещена** на бэкенде (жёсткая проверка), на фронте — кнопки скрыты или disabled с подсказкой.

Так мы не вводим второй параллельный «режим» флагом: один источник правды — поле `format`.

---

## Фаза 1 — База и защита API

1. **Prisma**  
   - В `TournamentFormat` добавить `MANUAL`.  
   - Миграция PostgreSQL: `ALTER TYPE ... ADD VALUE 'MANUAL'`.

2. **Точки запрета** (в начале метода — `BadRequestException` с понятным текстом):
   - `TournamentsService.generateCalendar`
   - `generateCalendarFromTemplate`
   - `generatePlayoff`
   - Любые другие методы, которые **массово создают** матчи по правилам турнира (пройтись по `tournaments.service.ts` и контроллеру).

3. **Опционально вспомогательная функция**  
   `assertNotManualFormat(tournament, 'Сообщение')` — чтобы не дублировать условие.

4. **DTO / валидация создания турнира**  
   - Разрешить `format: MANUAL` в `CreateTournamentDto` / `UpdateTournamentDto` (если enum генерируется из Prisma — обновить клиент после миграции).

## Фаза 2 — Фронт: форма турнира и карточка списка

1. **Список/форма турниров** (`admin/tournaments/index.vue`, общие константы в `tournamentAdminUi.ts` / `formatOptions`):
   - Пункт: например **«Только ручное расписание»** (`MANUAL`), краткое описание под полем.

2. **Страница турнира** `[id].vue`:
   - `generateCalendar`, кнопки шаблонов, «пересоздать» — **не показывать** или **disabled**, если `tournament.format === 'MANUAL'`.
   - Обновить `shouldRegenerateCalendar` / `canRegenerateCalendar` (или аналоги), чтобы для `MANUAL` всегда было «нельзя генерировать».

3. **Отображение формата** в списке/карточках — добавить человекочитаемую подпись для `MANUAL` в `formatLabel` / аналог.

## Фаза 3 — Ручные матчи

1. **Создание одного матча** — `POST` с `homeTeamId`, `awayTeamId`, `startTime` (+ опционально стадия/тур/группа при расширении). **Удаление** — `DELETE` по id матча.

2. Для `MANUAL` UI: «Добавить матч», список матчей с удалением; протокол счёта — как у остальных форматов.

---

## Фаза 4 — Смена формата (позже, по необходимости)

Продуктовые правила (выбрать одно и зафиксировать):

| Переход        | Варианты |
|----------------|----------|
| MANUAL → AUTO  | Запрет пока есть матчи / разрешить с предупреждением / только в `DRAFT` |
| AUTO → MANUAL  | Предупреждение: дальше без генерации; существующие матчи оставить |

Реализация: проверки в `PATCH /tournaments/:id` + подтверждение на фронте.

---

## Фаза 5 — Проверки и регресс

1. Ручной прогон: создать турнир `MANUAL` → убедиться, что все эндпоинты генерации отвечают 400.  
2. Существующие форматы (`SINGLE_GROUP`, …) — генерация как раньше.  
3. При необходимости — один e2e или unit-тест на guard в `generateCalendar`.

## Что сознательно откладываем

- Поле `source` у матча (`GENERATED` / `MANUAL`) — можно добавить позже, когда понадобится «перегенерировать только авто-матчи»; для чистого `MANUAL` турнира не обязательно с первого дня.
- Миграция старых данных — не требуется, если `MANUAL` только для новых турниров.
